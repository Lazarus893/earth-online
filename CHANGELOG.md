# Changelog

## [v4.0] - 2026-05-19 — 人本化升级

### 新增

- **AI 动态 Onboarding 对话**
  - 只有第 1 题固定（奇迹问题变体），后续由 AI 根据回答实时生成
  - 每轮结束后 AI 提取维度信号 + 生成即时反射（"我听到了"的感觉）
  - 回退策略：AI 不可用时自动使用关键词提取 + 备用问题池
  - 反射文案 8 种变体，避免重复

- **日记式任务系统 (JournalPanel)**
  - 替代原有 todo-list 风格的 QuestPanel
  - AI 从层级目标树中智能选取每日 3-5 个行动
  - "记录"而非"完成"，降低压迫感
  - 支持 AI 日记反思（完成过半后触发）

- **动态上下文引擎 (contextEngine)**
  - 场景感知：Dashboard / 维度详情 / 对话中自动切换 prompt 策略
  - 记忆种子：Onboarding 自由文本持久化，供后续 AI 调用引用
  - 交互摘要：每 5 条消息后台更新短期记忆
  - 预算感知：超过 token 限制时按优先级截断

- **Oracle 主动关怀系统**
  - 4 种触发场景：分数衰减（<30）、连续中断（曾 ≥7 天）、任务完成率低（连续 2 天 <50%）、维度停滞
  - 冷却机制：同类型 4 小时、每日最多 2 次
  - 点击通知 → 打开聊天 + 注入 Oracle 消息

- **维度状态文案**
  - 5 档 × 5 维度的描述性标签（如 "有点紧"、"状态不错"）
  - 状态颜色随分数动态变化

- **聊天面板清除按钮**
  - 带确认对话框，防误触

- **觉醒路径三选一视觉重构**
  - BEM 组件化设计
  - 悬停发光条 + 边框光晕 + 卡片浮动
  - 新手礼包展示区

### 修复

- 修复 checkbox 点击后不勾选的 bug（`logged` 字段未同步）
- 修复开场动画被 React Strict Mode 跳过的问题
- 修复选题后旧卡片闪现（loading 状态时序优化）
- 修复 DATA_VERSION 未更新导致旧数据残留

### 变更

- DATA_VERSION 从 `v6` 升级到 `v7`（首次使用会自动清理旧数据）
- Onboarding 从 6 题固定问卷改为 6 轮 AI 动态对话
- 任务面板从 QuestPanel 切换为 JournalPanel
- Chat system prompt 从硬编码改为动态上下文引擎生成
